# Sprint Plan – Library Management System

## Sprint Goal
Create basic features that allow users to search, borrow, and return books.

---

# User Stories

## 1. Search Books
User Story:
As a user, I want to search for books by title, so that I can quickly find the book I want.

Acceptance Criteria:
- User can type book title
- System shows matching books
- If no book exists, show "Book not found"

Priority: Must Have  
Story Points: 5  

Reason: Searching books is a core feature and requires some logic.

---

## 2. Borrow Book
User Story:
As a user, I want to borrow a book, so that I can read it.

Acceptance Criteria:
- User selects a book
- System marks book as borrowed
- Borrowed book cannot be borrowed again

Priority: Must Have  
Story Points: 8  

Reason: Borrowing books requires checking availability.

---

## 3. Return Book
User Story:
As a user, I want to return a borrowed book, so that other users can borrow it.

Acceptance Criteria:
- User selects borrowed book
- System updates book status to available

Priority: Must Have  
Story Points: 5  

Reason: Returning books updates the library system.

---

## 4. View Book List
User Story:
As a user, I want to see all books in the library, so that I know what books exist.

Acceptance Criteria:
- System displays list of books
- Shows title and author

Priority: Should Have  
Story Points: 3  

Reason: Easy feature but helpful.

---

## 5. Add New Book
User Story:
As a librarian, I want to add new books, so that the library collection can grow.

Acceptance Criteria:
- Librarian enters book title
- Book gets added to library list

Priority: Could Have  
Story Points: 3  

Reason: Small feature but not critical.

---

## 6. Remove Book
User Story:
As a librarian, I want to remove books, so that outdated books can be deleted.

Acceptance Criteria:
- Librarian selects book
- Book is removed from list

Priority: Won't Have  
Story Points: 2  

Reason: Not necessary for this sprint.
